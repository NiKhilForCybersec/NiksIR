CloudTrail Analysis

CloudTrail logs all API activity in AWS. It's your primary audit log for investigating who did what.

Key CloudTrail Fields

Field Description
eventTime When the action occurred
eventSource AWS service (ec2.amazonaws.com)
eventName API action (RunInstances, CreateUser)
userIdentity Who performed action (user, role, service)
sourceIPAddress Where the API call came from
userAgent Tool used (Console, CLI, SDK)
errorCode If failed, why (AccessDenied, etc.)
requestParameters Parameters of the API call

High-Value Events to Monitor

Event Risk Why Monitor
CreateUser, CreateAccessKey High Persistence, backdoor creation
AttachUserPolicy, PutUserPolicy High Privilege escalation
ConsoleLogin Medium Track console access
RunInstances Medium Resource creation (crypto mining)
StopLogging Critical Defense evasion
PutBucketPolicy, PutBucketAcl High Data exposure
AuthorizeSecurityGroupIngress Medium Network exposure

GuardDuty Findings

GuardDuty is AWS's threat detection service. It analyzes CloudTrail, VPC Flow Logs, and DNS logs to detect threats.

Finding Types

Recon Findings ‚ñº
  • Recon:IAMUser/... - API reconnaissance activity
  • Discovery:S3/... - S3 bucket enumeration

Indicates attacker gathering information about your environment.

UnauthorizedAccess Findings ‚ñº
  • UnauthorizedAccess:IAMUser/ConsoleLoginSuccess.B - Unusual console login
  • UnauthorizedAccess:IAMUser/InstanceCredentialExfiltration - Instance creds used outside EC2
  • UnauthorizedAccess:EC2/SSHBruteForce - SSH brute force
CryptoCurrency Findings ‚ñº
  • CryptoCurrency:EC2/BitcoinTool.B - Mining activity detected

Common indicator of compromised credentials or instances.

Trojan/Backdoor Findings ‚ñº
  • Trojan:EC2/... - Instance communicating with known malicious IPs
  • Backdoor:EC2/... - C2 communication detected
Stealth Findings ‚ñº
  • Stealth:IAMUser/CloudTrailLoggingDisabled - Logging disabled
  • Stealth:IAMUser/LoggingConfigurationModified - Logging tampered

Critical! Attacker trying to cover tracks.

Severity Levels

Severity Range Response
High 7.0 - 8.9 Immediate investigation
Medium 4.0 - 6.9 Investigate within 24 hours
Low 0.1 - 3.9 Review as time permits

Common AWS Threats

Credential Compromise

  • Access keys leaked in code
  • Phished console credentials
  • Stolen from compromised instances

Indicators: Unusual regions, new user agents, impossible travel

Crypto Mining

  • Large EC2 instances spun up
  • GPU instances in unusual regions
  • High CPU, network to mining pools

Indicators: GuardDuty CryptoCurrency findings

S3 Data Exposure

  • Public bucket policies
  • Misconfigured ACLs
  • Mass data downloads

Indicators: PutBucketPolicy, unusual GetObject volume

Privilege Escalation

  • Policy attachment to users
  • New admin users created
  • Role assumption abuse

Indicators: AttachUserPolicy, CreateUser, AssumeRole

Investigation Queries

Query Language Note

These queries are for CloudTrail logs in various formats. Adapt syntax for your SIEM (Sentinel, Splunk, native Athena, etc.).

KQL Console logins from unusual locations
AWSCloudTrail | where TimeGenerated > ago(24h) | where EventName == "ConsoleLogin" | where ResponseElements has "Success" | project TimeGenerated, UserIdentityUserName, SourceIpAddress, UserAgent, AWSRegion | order by TimeGenerated desc
KQL IAM user/access key creation
AWSCloudTrail | where TimeGenerated > ago(7d) | where EventName in ("CreateUser", "CreateAccessKey", "CreateLoginProfile") | project TimeGenerated, EventName, UserIdentityUserName, SourceIpAddress, RequestParameters | order by TimeGenerated desc
KQL EC2 instances launched
AWSCloudTrail | where TimeGenerated > ago(24h) | where EventName == "RunInstances" | extend InstanceType = tostring( parse_json(RequestParameters).instanceType) | project TimeGenerated, UserIdentityUserName, SourceIpAddress, AWSRegion, InstanceType | order by TimeGenerated desc
KQL S3 bucket policy changes
AWSCloudTrail | where TimeGenerated > ago(7d) | where EventSource == "s3.amazonaws.com" | where EventName in ("PutBucketPolicy", "PutBucketAcl", "PutBucketPublicAccessBlock") | project TimeGenerated, EventName, UserIdentityUserName, SourceIpAddress, RequestParameters
KQL Failed API calls (access denied)
AWSCloudTrail | where TimeGenerated > ago(24h) | where ErrorCode == "AccessDenied" | summarize FailedCount = count() by UserIdentityUserName, EventName, SourceIpAddress | where FailedCount > 5 | order by FailedCount desc
