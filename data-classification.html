Classification Levels

Data classification organizes information by sensitivity level, determining protection requirements and handling procedures.

Level Description Examples Protection
Highly Confidential Severe damage if disclosed Trade secrets, M&A data, credentials, source code Encryption, strict access, full audit
Confidential Significant damage if disclosed Customer PII, financial data, HR records Encryption, need-to-know access
Internal Not for public, limited impact Internal policies, org charts, project plans Internal access only
Public No impact if disclosed Marketing materials, public website No restrictions
Organization-Specific Labels

Your organization may use different labels (e.g., Restricted, Secret, Top Secret for government; or Tier 1/2/3). Learn your organization's specific classification scheme.

Regulatory Data Types

Certain data types have specific regulatory requirements that affect incident response.

Regulation Data Covered Key Requirements
PCI DSS Cardholder data (credit card numbers, CVV) Encryption, access controls, breach notification
HIPAA Protected Health Information (PHI) Privacy, security, breach notification
GDPR EU personal data 72-hour breach notification, right to erasure
CCPA/CPRA California resident personal data Disclosure rights, opt-out, breach notification
SOX Financial records Audit trails, access controls
GLBA Financial customer data Privacy notices, safeguards
Breach Notification Timelines

Regulations have strict notification timelines. GDPR requires 72-hour notification. If regulated data is involved in an incident, escalate immediately to Legal/Compliance.

SOC Analyst Role

What SOC Handles

SOC Responsibility

  • Triage DLP alerts
  • Initial investigation
  • Determine if data left organization
  • Identify affected data types
  • Escalate appropriately

Escalate To Others

  • Policy creation/tuning â†’ DLP Team
  • User discipline â†’ HR
  • Regulatory reporting â†’ Legal/Compliance
  • Data classification â†’ Data Owners
  • Technical remediation â†’ IT

Key Questions for Classification

  • What type of sensitive data was involved?
  • How many records were affected?
  • Is this regulated data (PCI, HIPAA, GDPR)?
  • Did data actually leave the organization?
  • Who was the intended/actual recipient?
  • What is the business impact?
Key Takeaways
  • Know your organization's classification levels
  • Regulated data (PCI, HIPAA, GDPR) requires immediate escalation
  • Volume matters: 1 record vs 1000 records = different response
  • Escalate to Legal/Compliance for any potential breach of regulated data
  • Document the data types involved in every DLP investigation
  • SOC triages and investigates; others handle policy and discipline
