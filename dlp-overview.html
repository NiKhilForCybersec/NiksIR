What is DLP?

Data Loss Prevention (DLP) is a set of technologies and processes designed to detect and prevent unauthorized transmission of sensitive data outside the organization.

Detect
Identify sensitive data
Monitor
Track data movement
Protect
Prevent unauthorized transfer
Report
Provide visibility
DLP vs. Encryption

Encryption protects data if it's stolen (renders it unreadable).
DLP prevents data from leaving in the first place. Both work together for defense in depth.

DLP Types

Endpoint DLP ‚ñº

Monitors and controls data on user devices.

Channels Covered:

  • USB/removable storage
  • Clipboard (copy/paste)
  • Print to local/network printers
  • Upload to cloud services
  • Application access (browser, apps)

Vendors: Microsoft Purview Endpoint DLP, Symantec DLP, Digital Guardian, Forcepoint

Network DLP ‚ñº

Monitors data in transit on the network.

Channels Covered:

  • Email (SMTP)
  • Web traffic (HTTP/HTTPS with inspection)
  • FTP/SFTP
  • IM/collaboration tools

Vendors: Symantec DLP Network, Forcepoint, Netskope (inline)

Cloud DLP ‚ñº

Protects data in cloud applications and storage.

Channels Covered:

  • Cloud storage (OneDrive, SharePoint, Box, Dropbox)
  • SaaS applications (Salesforce, Workday)
  • Cloud email (Exchange Online, Gmail)
  • Collaboration (Teams, Slack)

Vendors: Microsoft Purview, Netskope, Zscaler, Palo Alto Prisma

DLP Actions

Audit Only

Log activity but take no action. Used for:

  • Testing new policies
  • Understanding data flows
  • Building baseline

Warn (Policy Tip)

Notify user but allow action. Used for:

  • User education
  • Low-risk scenarios
  • Business justification collection

Block with Override

Block by default, allow with justification:

  • Legitimate business need possible
  • Manager approval flow
  • Audit trail of overrides

Block

Prevent action completely:

  • High-risk data types
  • Regulatory requirements
  • No legitimate external use
Enforcement Progression

Most organizations start with Audit ‚Üí Warn ‚Üí Block with Override ‚Üí Block to minimize business disruption and gather feedback before full enforcement.

SOC Analyst Role in DLP

What SOC Typically Handles

Activity SOC Role
High-severity DLP alerts Investigate and escalate potential data breaches
Exfiltration patterns Correlate DLP with other indicators (unusual volume, after-hours)
Insider threat indicators Investigate users with repeated violations
Incident response Assess data impact during security incidents

What Compliance/DLP Team Typically Handles

Activity Owner
Policy creation and tuning DLP/Compliance Team
Low-severity policy violations DLP/Compliance Team
User education DLP/Compliance Team
False positive tuning DLP/Compliance Team
Regulatory reporting DLP/Compliance/Legal
Know Your Escalation Path

DLP incidents often require coordination with Legal, HR, and Compliance teams. Know who to contact before you need to‚Äîespecially for potential data breaches involving regulated data (PII, PHI, PCI).

Key Takeaways
  • DLP detects and prevents sensitive data from leaving the organization
  • Three main types: Endpoint, Network, Cloud
  • Actions range from Audit ‚Üí Warn ‚Üí Block with Override ‚Üí Block
  • SOC focuses on high-severity alerts and exfiltration patterns
  • DLP/Compliance team handles policy tuning and low-severity events
  • Know escalation paths for data breaches involving regulated data
