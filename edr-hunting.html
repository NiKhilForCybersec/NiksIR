EDR Hunting Approach

EDR platforms provide rich telemetry for proactive threat hunting. Unlike reactive alert investigation, hunting seeks threats that evaded detection.

Hunting Data Sources

Data Type What to Hunt
Process Events Suspicious parent-child, unusual commands
Network Events Beaconing, unusual destinations, C2 patterns
File Events Suspicious writes, script drops, staging
Registry Events Persistence mechanisms
Authentication Credential use patterns

Hunt Types

Anomaly Hunting

Find outliers and unusual patterns

  • Rare processes
  • First-time executions
  • Statistical anomalies

TTP Hunting

Search for known attack techniques

  • MITRE ATT&CK patterns
  • Known tool signatures
  • Attack chain indicators

Process Hunting

Suspicious Process Patterns

Pattern Indicates
Office â†’ cmd/PowerShell Macro execution
Browser â†’ script interpreter Drive-by download
svchost from wrong path Process masquerading
Rare process, many hosts Potential malware spread
PowerShell -enc with long string Encoded commands
KQL Rare processes across environment
// Find processes seen on very few devices DeviceProcessEvents | where TimeGenerated > ago(7d) | summarize DeviceCount = dcount(DeviceName), Devices = make_set(DeviceName, 5), ExecutionCount = count() by FileName, FolderPath | where DeviceCount == 1 and ExecutionCount < 5 | order by ExecutionCount asc
KQL First-time process executions
// Processes seen for first time in last 24h let Baseline = DeviceProcessEvents | where TimeGenerated between (ago(30d) .. ago(1d)) | distinct FileName, SHA256; DeviceProcessEvents | where TimeGenerated > ago(24h) | join kind=leftanti Baseline on FileName, SHA256 | summarize FirstSeen = min(TimeGenerated), DeviceCount = dcount(DeviceName) by FileName, SHA256 | order by DeviceCount desc

Network Hunting

Beaconing Detection

C2 beacons often have regular intervals. Hunt for processes with consistent connection timing.

KQL Potential beaconing (regular intervals)
DeviceNetworkEvents | where TimeGenerated > ago(24h) | where RemoteIPType == "Public" | summarize Connections = make_list(TimeGenerated) by DeviceName, InitiatingProcessFileName, RemoteIP | where array_length(Connections) > 10 | extend Intervals = series_stats_dynamic( series_subtract(Connections, array_shift_left(Connections, 1)) ) | extend AvgInterval = Intervals.avg, StdDev = Intervals.stdev | extend CoeffVariation = StdDev / AvgInterval | where CoeffVariation < 0.3 // Low variation = regular beaconing | project DeviceName, InitiatingProcessFileName, RemoteIP, AvgInterval, CoeffVariation

Unusual Destinations

KQL Connections to rare external IPs
DeviceNetworkEvents | where TimeGenerated > ago(7d) | where RemoteIPType == "Public" | summarize DeviceCount = dcount(DeviceName), Devices = make_set(DeviceName, 5), Processes = make_set(InitiatingProcessFileName, 5) by RemoteIP | where DeviceCount == 1 // Only one device connected | order by DeviceCount asc

File Hunting

Suspicious File Locations

Location Why Suspicious
C:\Users\Public\ Writable by all, malware staging
C:\ProgramData\ Hidden folder, persistence
%TEMP% Execution from temp
Root of C:\ Unusual executable location
Recycle Bin Hidden execution
KQL Executables in suspicious locations
DeviceFileEvents | where TimeGenerated > ago(7d) | where ActionType == "FileCreated" | where FileName endswith ".exe" or FileName endswith ".dll" | where FolderPath has_any ( @"\Users\Public\", @"\Temp\", @"\$Recycle.Bin\", @"C:\" // Root of C: only ) | project TimeGenerated, DeviceName, FileName, FolderPath, InitiatingProcessFileName, SHA256

Additional Hunting Queries

KQL Scripts executed from user-writable locations
DeviceProcessEvents | where TimeGenerated > ago(7d) | where FileName in~ ("powershell.exe", "cmd.exe", "wscript.exe", "cscript.exe") | where ProcessCommandLine has_any ( @"\Users\", @"\Temp\", @"\Downloads\", @"\AppData\" ) | where ProcessCommandLine has_any (".ps1", ".vbs", ".bat", ".js") | project TimeGenerated, DeviceName, AccountName, FileName, ProcessCommandLine
KQL Processes with no parent (potential injection)
DeviceProcessEvents | where TimeGenerated > ago(7d) | where isempty(InitiatingProcessFileName) | where FileName !in~ ("System", "smss.exe") // Expected | project TimeGenerated, DeviceName, FileName, ProcessCommandLine, AccountName
