Initial Triage Questions

  • What triggered the alert? Specific detection logic
  • What user/device? Context about normal activity
  • What process? Legitimate or suspicious?
  • What was the parent process? How did it start?
  • What time? Business hours or unusual?
  • Has this happened before? Check for patterns

Process Analysis

Process tree analysis is the core of EDR investigation. Understanding parent-child relationships reveals attack chains.

Process Tree Red Flags

Suspicious Parent-Child Relationships ‚ñº
Parent Suspicious Child Likely Attack
WINWORD.EXE / EXCEL.EXE cmd.exe, powershell.exe Macro execution
OUTLOOK.EXE powershell.exe, cmd.exe Malicious attachment/link
browser (chrome, edge) powershell.exe, cmd.exe Drive-by download
wmiprvse.exe powershell.exe WMI lateral movement
services.exe cmd.exe (unusual) Service exploitation
svchost.exe powershell.exe Scheduled task abuse
Suspicious Command Line Patterns ‚ñº

PowerShell Red Flags:

  • -enc or -encodedcommand - Base64 encoded
  • -nop or -noprofile - Skip profile
  • -w hidden - Hidden window
  • -ep bypass - Execution policy bypass
  • IEX or Invoke-Expression - Execute string
  • DownloadString / DownloadFile - Download payload
  • Net.WebClient - Web download

CMD Red Flags:

  • Long obfuscated commands
  • Multiple && or || chained commands
  • certutil with -urlcache or -decode
  • bitsadmin /transfer
Process Masquerading ‚ñº

Attackers disguise malware as legitimate processes:

  • Wrong path: svchost.exe not in System32
  • Typosquatting: svch0st.exe, scvhost.exe
  • Wrong parent: svchost.exe not from services.exe
  • Unsigned: Windows binaries should be signed

Expected Paths:

svchost.exe    ‚Üí C:\Windows\System32\
lsass.exe      ‚Üí C:\Windows\System32\
csrss.exe      ‚Üí C:\Windows\System32\
services.exe   ‚Üí C:\Windows\System32\
smss.exe       ‚Üí C:\Windows\System32\
KQL Build process tree for suspicious process
let TargetDevice = "WORKSTATION-01"; let SuspiciousPID = 1234; let TimeWindow = ago(24h); DeviceProcessEvents | where TimeGenerated > TimeWindow | where DeviceName =~ TargetDevice | where ProcessId == SuspiciousPID or InitiatingProcessId == SuspiciousPID or ProcessId == InitiatingProcessId | project TimeGenerated, ProcessId, FileName, ProcessCommandLine, InitiatingProcessId, InitiatingProcessFileName, InitiatingProcessCommandLine, AccountName | order by TimeGenerated asc

Network Analysis

Network activity from suspicious processes can reveal C2 communication, data exfiltration, or lateral movement.

Network Red Flags

  • Connections to unusual countries
  • High data transfer to external IPs
  • Connections on non-standard ports
  • Process that shouldn't make network connections (notepad, calc)
  • Regular interval connections (beaconing)
  • DNS queries to suspicious domains
  • Direct IP connections (no DNS)
KQL Network activity for suspicious process
let TargetDevice = "WORKSTATION-01"; let SuspiciousProcess = "powershell.exe"; let TimeWindow = ago(24h); DeviceNetworkEvents | where TimeGenerated > TimeWindow | where DeviceName =~ TargetDevice | where InitiatingProcessFileName =~ SuspiciousProcess | summarize ConnectionCount = count(), TotalBytesSent = sum(SentBytes), TotalBytesReceived = sum(ReceivedBytes), Ports = make_set(RemotePort), URLs = make_set(RemoteUrl, 10) by RemoteIP, RemoteIPType | order by ConnectionCount desc
KQL Check IP reputation and connection timing
let SuspiciousIP = "203.0.113.50"; DeviceNetworkEvents | where TimeGenerated > ago(7d) | where RemoteIP == SuspiciousIP | project TimeGenerated, DeviceName, InitiatingProcessFileName, InitiatingProcessCommandLine, RemotePort, SentBytes, ReceivedBytes | order by TimeGenerated asc

File Analysis

File events reveal malware drops, staging files, and exfiltration preparation.

File Activity Red Flags

  • Executables in temp folders
  • Files with double extensions (.pdf.exe)
  • Recently created files in startup locations
  • Large files created before network upload
  • Many files renamed with same extension (ransomware)
  • Script files (.ps1, .vbs, .bat) in unusual locations
KQL Files created by suspicious process
let TargetDevice = "WORKSTATION-01"; let SuspiciousProcess = "powershell.exe"; DeviceFileEvents | where TimeGenerated > ago(24h) | where DeviceName =~ TargetDevice | where InitiatingProcessFileName =~ SuspiciousProcess | where ActionType in ("FileCreated", "FileModified") | project TimeGenerated, ActionType, FileName, FolderPath, SHA256, FileSize, InitiatingProcessCommandLine | order by TimeGenerated asc

Hash Investigation

Check file hashes against threat intelligence:

  • VirusTotal: Multi-engine scan results
  • Hybrid Analysis: Behavioral analysis
  • Internal IOC database: Known bad hashes

Timeline Building

Combine process, network, and file events into a unified timeline to understand the full attack sequence.

KQL Unified timeline for device
let TargetDevice = "WORKSTATION-01"; let TimeWindow = ago(24h); let ProcessEvents = DeviceProcessEvents | where TimeGenerated > TimeWindow | where DeviceName =~ TargetDevice | project TimeGenerated, EventType = "Process", Description = strcat(FileName, " - ", substring(ProcessCommandLine, 0, 100)), Details = InitiatingProcessFileName; let NetworkEvents = DeviceNetworkEvents | where TimeGenerated > TimeWindow | where DeviceName =~ TargetDevice | project TimeGenerated, EventType = "Network", Description = strcat(RemoteIP, ":", RemotePort), Details = InitiatingProcessFileName; let FileEvents = DeviceFileEvents | where TimeGenerated > TimeWindow | where DeviceName =~ TargetDevice | where ActionType in ("FileCreated", "FileModified") | project TimeGenerated, EventType = "File", Description = strcat(ActionType, ": ", FileName), Details = FolderPath; union ProcessEvents, NetworkEvents, FileEvents | order by TimeGenerated asc

Response Actions

EDR Response Capabilities

Action Description When to Use
Isolate Device Cut network except EDR communication Confirmed compromise, prevent spread
Kill Process Terminate running malicious process Active malware execution
Quarantine File Move malicious file to quarantine Identified malware file
Block Hash Prevent file from running anywhere Known malicious file
Collect Investigation Package Gather forensic artifacts Need deeper analysis
Run Live Response Remote shell/commands on endpoint Manual investigation/remediation

Response Decision Tree

  • Confirmed malware execution? ‚Üí Isolate, kill process, collect artifacts
  • Suspicious but not confirmed? ‚Üí Collect investigation package, continue analysis
  • Lateral movement detected? ‚Üí Isolate affected systems, hunt for spread
  • Data exfiltration? ‚Üí Isolate, preserve evidence, escalate to IR
  • False positive? ‚Üí Document, consider tuning
Key Takeaways
  • Start with process tree‚Äîparent-child relationships tell the story
  • Office apps spawning cmd/PowerShell is almost always malicious
  • Check command line for encoded commands, bypass flags, downloads
  • Verify process paths‚Äîsvchost outside System32 is suspicious
  • Correlate network activity with process for C2 identification
  • Build timeline combining process, network, and file events
  • Isolate before it spreads if confirmed compromise
