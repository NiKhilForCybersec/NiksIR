GCP Logging

Google Cloud provides comprehensive logging through Cloud Audit Logs and Cloud Logging. Understanding these is essential for security monitoring.

Log Types

Log Type What It Captures Default Enabled
Admin Activity Config changes, resource modifications Yes
Data Access Resource reads, data access No (enable)
System Event Google-initiated actions Yes
Policy Denied Access denied by policy Yes
Enable Data Access Logs

Data Access logs are critical for security but disabled by default. Enable them for sensitive resources (BigQuery, Cloud Storage, etc.) to detect data exfiltration.

Key Log Fields

Field Description
protoPayload.methodName API method called (action taken)
protoPayload.authenticationInfo.principalEmail Who performed the action
protoPayload.requestMetadata.callerIp Source IP address
resource.type Type of resource affected
protoPayload.serviceName GCP service (compute, storage, etc.)

Security Command Center

Security Command Center (SCC) is GCP's native security management platform.

SCC Components

Feature Function
Security Health Analytics Misconfigurations, vulnerabilities
Event Threat Detection Threat detection from logs
Container Threat Detection GKE runtime threats
Web Security Scanner App vulnerability scanning

Event Threat Detection Findings

Finding Type What It Detects
Malware Cryptomining, malicious binaries
Persistence SSH key additions, IAM grants
Credential Access Service account key creation
Evasion Audit logging disabled
Exfiltration Data copied to external location

Common GCP Threats

Service Account Key Theft ‚ñº

Threat:

Service account keys leaked in code, logs, or stolen from metadata.

Indicators:

  • New service account key created
  • Service account used from unusual IP
  • Key downloaded

Detection:

Monitor CreateServiceAccountKey, unusual authentication patterns.

Public Cloud Storage ‚ñº

Threat:

Cloud Storage buckets made publicly accessible.

Indicators:

  • SetIamPolicy with allUsers or allAuthenticatedUsers
  • Public access grants

Detection:

Alert on IAM policy changes granting public access.

Compute Instance Compromise ‚ñº

Threat:

VMs compromised for cryptomining or other abuse.

Indicators:

  • High CPU GPU-intensive instances
  • Connections to mining pools
  • Unusual instance creation

Detection:

SCC cryptomining findings, billing alerts, VPC flow logs.

IAM Privilege Escalation ‚ñº

Threat:

Attacker escalates from limited to admin permissions.

Indicators:

  • SetIamPolicy granting owner/editor roles
  • Service account impersonation
  • New project-level bindings

Detection:

Monitor IAM policy changes, especially to privileged roles.

Key Events to Monitor

Method Risk Why It Matters
CreateServiceAccountKey High Persistent credential created
SetIamPolicy High Permission changes
compute.instances.insert Medium New VM created (cryptomining)
compute.firewalls.insert Medium Firewall rule added
storage.setIamPermissions High Bucket permissions changed
DeleteSink (logging) Critical Audit logging disrupted

Investigation Queries

Cloud Logging Query Syntax

Cloud Logging Queries 
# Service account key creation
protoPayload.methodName="google.iam.admin.v1.CreateServiceAccountKey"

# IAM policy changes
protoPayload.methodName="SetIamPolicy"

# Failed authentication attempts
protoPayload.status.code="7"  -- PERMISSION_DENIED

# Activity by specific user
protoPayload.authenticationInfo.principalEmail="user@domain.com"

# Compute instance creation
protoPayload.methodName="v1.compute.instances.insert"

# Public bucket access
protoPayload.methodName="storage.setIamPermissions"
protoPayload.serviceData.policyDelta.bindingDeltas.member="allUsers"
KQL GCP high-risk events (if logs in Sentinel)
GCPAuditLogs | where TimeGenerated > ago(24h) | where MethodName in ( "google.iam.admin.v1.CreateServiceAccountKey", "SetIamPolicy", "v1.compute.instances.insert" ) | project TimeGenerated, MethodName, PrincipalEmail, CallerIp, ResourceName | order by TimeGenerated desc
Key Takeaways
  • Enable Data Access logs for sensitive resources
  • Service account key creation is high-risk‚Äîmonitor closely
  • SetIamPolicy changes can lead to privilege escalation
  • Use Security Command Center for threat detection
  • Watch for compute instance creation (cryptomining)
  • Alert on public access grants to Cloud Storage
  • Monitor for logging sink deletion (covering tracks)
