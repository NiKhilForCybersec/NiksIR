Why Identity Matters

Identity is the new perimeter. With cloud adoption and remote work, attackers increasingly target identities rather than network boundaries. Compromising one privileged account can give access to the entire environment.

Identity-First Security

Modern security assumes the network is already breached. Identity verification becomes the primary controlâ€”every access request must be authenticated, authorized, and continuously validated.

Authentication vs Authorization

Authentication (AuthN)

"Who are you?"

Verifying the identity of a user or system.

Factors:
  • Something you know: Password, PIN
  • Something you have: Phone, token, smart card
  • Something you are: Fingerprint, face, voice

Authorization (AuthZ)

"What can you do?"

Determining what resources an authenticated user can access.

Methods:
  • RBAC: Role-Based Access Control
  • ABAC: Attribute-Based Access Control
  • ACLs: Access Control Lists

Identity Providers

Active Directory (On-Premises) â–¼

Microsoft's directory service for Windows domain networks.

Key Components:

  • Domain Controllers (DC): Authenticate users, store directory
  • Kerberos: Primary authentication protocol
  • NTLM: Legacy authentication (should be minimized)
  • Group Policy: Centralized configuration management
  • LDAP: Directory query protocol

Critical Objects:

  • Domain Admins, Enterprise Admins, Schema Admins groups
  • KRBTGT account (Kerberos ticket granting)
  • Service accounts with SPNs
Azure AD / Microsoft Entra ID (Cloud) â–¼

Microsoft's cloud identity and access management service.

Key Features:

  • Single Sign-On (SSO): One identity for all cloud apps
  • Conditional Access: Context-aware access policies
  • Identity Protection: Risk-based detection
  • Privileged Identity Management: Just-in-time admin access
  • B2B/B2C: External identity management

Critical Roles:

  • Global Administrator
  • Privileged Role Administrator
  • Exchange Administrator
  • Application Administrator
Hybrid Identity â–¼

Most enterprises use both on-prem AD and Azure AD together.

Sync Methods:

  • Azure AD Connect: Syncs on-prem AD to Azure AD
  • Password Hash Sync (PHS): Hashes synced to cloud
  • Pass-Through Auth (PTA): Auth stays on-prem
  • Federation (ADFS): On-prem handles all auth
Hybrid Risk

Compromise of on-prem AD can lead to Azure AD compromise through sync. Azure AD Connect servers are high-value targets.

Common Identity Attacks

Attack Description Detection
Credential Phishing Fake login pages to steal passwords Email gateway, user reports, impossible travel
Password Spraying Try common passwords against many accounts Many failed logins across accounts, same password
Credential Stuffing Use leaked credentials from other breaches Failed logins from known bad IPs, unusual locations
Brute Force Try many passwords against one account Many failed logins for single account
MFA Fatigue Spam push notifications until user approves Many MFA prompts in short time, after-hours
AiTM Phishing Proxy attack captures session tokens Sign-in from phishing infra, unusual session
Kerberoasting Request service tickets, crack offline High volume of TGS requests for SPNs
Golden Ticket Forged Kerberos TGT using KRBTGT hash Tickets with long lifetimes, impossible access
Pass-the-Hash Use stolen NTLM hash to authenticate NTLM auth from unexpected sources
Token Theft Steal session/refresh tokens from device Token use from different device/location

What to Monitor

High Priority

  • Failed logins to privileged accounts
  • Logins from unusual locations
  • Impossible travel alerts
  • Admin role assignments
  • MFA registration changes
  • Conditional Access policy changes

Medium Priority

  • Password changes/resets
  • New device registrations
  • OAuth app consents
  • Guest user additions
  • Legacy authentication usage
  • Service principal activity

Investigate

  • After-hours privileged access
  • Bulk user enumeration
  • Password spray patterns
  • Unusual application access
  • Cross-tenant activity
KQL Risky sign-ins in Azure AD
SigninLogs | where TimeGenerated > ago(24h) | where RiskLevelDuringSignIn in ("high", "medium") | project TimeGenerated, UserPrincipalName, IPAddress, Location, AppDisplayName, RiskLevelDuringSignIn, RiskState, DeviceDetail | order by TimeGenerated desc

Identity Investigation Focus

When investigating identity-related alerts, answer these questions:

  • Who: Which account? What privileges does it have?
  • From where: IP address, location, deviceâ€”are they expected?
  • When: Normal working hours? Expected timing?
  • What: What did the account access after authentication?
  • How: What authentication method? Legacy auth? MFA bypassed?
  • Pattern: Is this consistent with user's normal behavior?
Quick Validation

For suspicious sign-ins, contact the user through a trusted channel (phone call, in-person) to verify if they performed the activity. Don't use emailâ€”it may be compromised.

Key Takeaways
  • Identity is the primary attack surface in modern environments
  • Authentication = who you are; Authorization = what you can do
  • MFA significantly reduces risk but isn't foolproof (AiTM, fatigue)
  • Monitor privileged accounts and administrative changes closely
  • Hybrid environments have compounded attack surface
  • Verify suspicious activity through out-of-band communication
