Join Operations

Joins combine data from multiple tables. Understanding join types is critical for correlating events across data sources.

Join Types

Type Returns Use Case
inner Only matching rows from both tables Find events that exist in both sources
leftouter All left rows + matching right Enrich events with optional data
rightouter All right rows + matching left Reverse of leftouter
fullouter All rows from both tables Complete union with matching
leftanti Left rows with NO match in right Find events NOT in reference set
leftsemi Left rows that HAVE match in right Filter by existence in another table
KQL Inner join: Correlate sign-ins with endpoint events
// Find sign-ins followed by suspicious process execution let SuspiciousSignIns = SigninLogs | where TimeGenerated > ago(1h) | where RiskLevelDuringSignIn == "high" | project SignInTime = TimeGenerated, UserPrincipalName, IPAddress; DeviceProcessEvents | where TimeGenerated > ago(1h) | join kind=inner SuspiciousSignIns on $left.AccountUpn == $right.UserPrincipalName | where TimeGenerated > SignInTime // Process after sign-in | where TimeGenerated < SignInTime + 30m // Within 30 min | project SignInTime, TimeGenerated, UserPrincipalName, DeviceName, FileName, ProcessCommandLine
KQL Leftanti join: Find new processes not in baseline
// Baseline: processes seen in last 30 days let Baseline = DeviceProcessEvents | where TimeGenerated between (ago(30d) .. ago(1d)) | distinct SHA256; // Find processes in last 24h NOT in baseline DeviceProcessEvents | where TimeGenerated > ago(1d) | join kind=leftanti Baseline on SHA256 | summarize FirstSeen = min(TimeGenerated), ExecutionCount = count(), Devices = dcount(DeviceName) by FileName, SHA256, FolderPath | order by ExecutionCount desc
KQL Leftouter join: Enrich alerts with threat intelligence
// Assume ThreatIntelligenceIndicator contains known bad IPs DeviceNetworkEvents | where TimeGenerated > ago(24h) | where RemoteIPType == "Public" | join kind=leftouter ( ThreatIntelligenceIndicator | where isnotempty(NetworkIP) | project ThreatIP = NetworkIP, ThreatType, Description, ConfidenceScore ) on $left.RemoteIP == $right.ThreatIP | where isnotempty(ThreatType) // Only show matches | project TimeGenerated, DeviceName, RemoteIP, ThreatType, Description, ConfidenceScore
Join Performance Tips
  • Always filter tables BEFORE joining (reduce data first)
  • Put the smaller table on the right side
  • Use project to include only needed columns
  • Avoid joining on string columns when possibleâ€”use hashes

Time Series Analysis

Time series functions help identify patterns, trends, and anomalies over time.

Key Functions

Function Purpose
make-series Create time series array from events
series_decompose Separate trend, seasonality, residual
series_decompose_anomalies Detect anomalies in time series
series_stats Calculate statistics (min, max, avg, stdev)
series_outliers Find outlier points in series
KQL Create time series of login failures
SigninLogs | where TimeGenerated > ago(7d) | where ResultType != "0" | make-series FailedLogins = count() on TimeGenerated from ago(7d) to now() step 1h | render timechart
KQL Detect anomalies in outbound traffic volume
DeviceNetworkEvents | where TimeGenerated > ago(14d) | where RemoteIPType == "Public" | summarize BytesSent = sum(SentBytes) by DeviceName, bin(TimeGenerated, 1h) | make-series Traffic = sum(BytesSent) on TimeGenerated from ago(14d) to now() step 1h by DeviceName | extend (anomalies, score, baseline) = series_decompose_anomalies(Traffic, 1.5) | mv-expand TimeGenerated, Traffic, anomalies, score | where anomalies == 1 // Positive anomaly (spike) | project TimeGenerated, DeviceName, Traffic = tolong(Traffic), AnomalyScore = todouble(score) | order by AnomalyScore desc
KQL Baseline comparison: this week vs last week
let ThisWeek = SecurityEvent | where TimeGenerated > ago(7d) | summarize CurrentCount = count() by EventID; let LastWeek = SecurityEvent | where TimeGenerated between (ago(14d) .. ago(7d)) | summarize BaselineCount = count() by EventID; ThisWeek | join kind=leftouter LastWeek on EventID | extend BaselineCount = coalesce(BaselineCount, 0) | extend PercentChange = round( (CurrentCount - BaselineCount) * 100.0 / BaselineCount, 2) | where abs(PercentChange) > 50 // >50% change | order by abs(PercentChange) desc

Anomaly Detection

KQL Statistical anomaly: Login times outside normal hours
// Build user login time baseline let UserBaseline = SigninLogs | where TimeGenerated between (ago(30d) .. ago(1d)) | where ResultType == "0" | extend HourOfDay = datetime_part("hour", TimeGenerated) | summarize AvgHour = avg(HourOfDay), StdevHour = stdev(HourOfDay), LoginCount = count() by UserPrincipalName | where LoginCount > 10; // Need enough data // Find logins outside 2 standard deviations SigninLogs | where TimeGenerated > ago(1d) | where ResultType == "0" | extend HourOfDay = datetime_part("hour", TimeGenerated) | join kind=inner UserBaseline on UserPrincipalName | extend ZScore = (HourOfDay - AvgHour) / StdevHour | where abs(ZScore) > 2 // Outside 2 stdev | project TimeGenerated, UserPrincipalName, HourOfDay, AvgHour = round(AvgHour, 1), ZScore = round(ZScore, 2), IPAddress, Location
KQL Peer group anomaly: User accessing unusual resources
// What resources do users in the same department access? let DepartmentBaseline = OfficeActivity | where TimeGenerated > ago(30d) | where Operation == "FileAccessed" | extend Department = tostring(split(UserId, "@")[0]) // Simplified | summarize AccessCount = count() by Department, OfficeObjectId | where AccessCount > 5; // Common resources // Find users accessing resources their peers don't OfficeActivity | where TimeGenerated > ago(1d) | where Operation == "FileAccessed" | extend Department = tostring(split(UserId, "@")[0]) | join kind=leftanti DepartmentBaseline on Department, OfficeObjectId | summarize UnusualAccess = count() by UserId, OfficeObjectId | where UnusualAccess > 3
KQL Beaconing detection with coefficient of variation
// Detect regular interval connections (C2 beaconing) DeviceNetworkEvents | where TimeGenerated > ago(24h) | where RemoteIPType == "Public" | summarize Connections = make_list(TimeGenerated) by DeviceName, RemoteIP | where array_length(Connections) > 20 // Enough data points | mv-apply Connections to typeof(datetime) on ( sort by Connections asc | extend NextConn = next(Connections) | extend Interval = datetime_diff('second', NextConn, Connections) | where isnotnull(Interval) | summarize AvgInterval = avg(Interval), StdevInterval = stdev(Interval), ConnectionCount = count() ) | extend CoeffOfVariation = StdevInterval / AvgInterval | where CoeffOfVariation < 0.2 // Low variation = regular intervals | where AvgInterval between (30 .. 3600) // 30s to 1hr typical C2 | project DeviceName, RemoteIP, ConnectionCount, AvgIntervalSec = round(AvgInterval, 0), CoeffOfVariation = round(CoeffOfVariation, 3) | order by CoeffOfVariation asc

Advanced Parsing

KQL Parse complex command lines
DeviceProcessEvents | where TimeGenerated > ago(24h) | where FileName =~ "powershell.exe" | extend CmdLine = ProcessCommandLine // Extract encoded command | extend EncodedCmd = extract(@"-[eE](?:nc|ncodedcommand)?\s+([A-Za-z0-9+/=]+)", 1, CmdLine) // Extract download URLs | extend DownloadURL = extract(@"https?://[^\s'\"]+", 0, CmdLine) // Check for bypass flags | extend HasBypass = CmdLine has_any ("-bypass", "-ep bypass", "executionpolicy bypass") | extend HasHidden = CmdLine has_any ("-w hidden", "-windowstyle hidden") | where isnotempty(EncodedCmd) or isnotempty(DownloadURL) or HasBypass or HasHidden | project TimeGenerated, DeviceName, AccountName, EncodedCmd, DownloadURL, HasBypass, HasHidden
KQL Parse JSON from string fields
AuditLogs | where TimeGenerated > ago(24h) | where OperationName == "Update user" | mv-expand TargetResources | mv-expand ModifiedProperties = TargetResources.modifiedProperties | extend PropertyName = tostring(ModifiedProperties.displayName) | extend OldValue = tostring(ModifiedProperties.oldValue) | extend NewValue = tostring(ModifiedProperties.newValue) | where PropertyName in ("AccountEnabled", "StrongAuthenticationRequirements") | project TimeGenerated, InitiatedBy = InitiatedBy.user.userPrincipalName, TargetUser = TargetResources.userPrincipalName, PropertyName, OldValue, NewValue
KQL Dynamic parsing with parse operator
// Parse syslog message format Syslog | where TimeGenerated > ago(1h) | where Facility == "auth" | parse SyslogMessage with * "user=" Username: string " " * "src=" SourceIP: string " " * "action=" Action: string | where isnotempty(Username) | project TimeGenerated, Computer, Username, SourceIP, Action

Functions & Variables

KQL Reusable function for process tree
// Define function to get process ancestry let GetProcessTree = (targetDevice: string, targetPID: long, lookbackHours: int) { DeviceProcessEvents | where TimeGenerated > ago(lookbackHours * 1h) | where DeviceName =~ targetDevice | where ProcessId == targetPID or InitiatingProcessId == targetPID | project TimeGenerated, ProcessId, FileName, ProcessCommandLine, InitiatingProcessId, InitiatingProcessFileName, InitiatingProcessCommandLine, AccountName }; // Use the function GetProcessTree("WORKSTATION-01", 1234, 24) | order by TimeGenerated asc
KQL Dynamic watchlists and IOC matching
// Define IOCs as dynamic arrays let BadIPs = dynamic([ "192.168.1.100", "10.0.0.50", "203.0.113.5" ]); let BadDomains = dynamic([ "evil.com", "malware.net", "c2server.org" ]); let BadHashes = dynamic([ "abc123...", "def456..." ]); // Check network events against IOCs DeviceNetworkEvents | where TimeGenerated > ago(24h) | where RemoteIP in (BadIPs) or RemoteUrl has_any (BadDomains) | project TimeGenerated, DeviceName, RemoteIP, RemoteUrl, InitiatingProcessFileName, InitiatingProcessCommandLine

Performance Optimization

âœ“ Do This

  • Filter on time first (where TimeGenerated)
  • Use has instead of contains
  • Use =~ for case-insensitive only when needed
  • Project early to reduce columns
  • Filter before joins
  • Use summarize to reduce rows

âœ— Avoid This

  • Querying without time filter
  • Using * wildcard unnecessarily
  • contains on large string columns
  • Regex when simpler operators work
  • Joining large tables without filtering
  • Multiple extend when one works
KQL Optimized vs unoptimized comparison
// âŒ SLOW - Unoptimized SecurityEvent | where Computer contains "SERVER" // Full string scan | where TimeGenerated > ago(7d) // Time filter too late | extend LowerAccount = tolower(Account) // Unnecessary transform | extend Hour = datetime_part("hour", TimeGenerated) // âœ“ FAST - Optimized SecurityEvent | where TimeGenerated > ago(7d) // Time first! | where Computer has "SERVER" // has is faster | extend Hour = datetime_part("hour", TimeGenerated) // Only what's needed

Query Execution Order

KQL executes operators in order. Structure queries for efficiency:

  1. Time filter - Reduces data volume immediately
  2. Table filter - Further reduce before processing
  3. Project - Remove unneeded columns
  4. Extend/Parse - Add calculated fields
  5. Summarize - Aggregate data
  6. Join - Combine with other data
  7. Order/Render - Final presentation

Threat Hunting Patterns

KQL Hunt: Rare parent-child process relationships
// Find unusual parent spawning unusual child let ProcessRelationships = DeviceProcessEvents | where TimeGenerated > ago(7d) | summarize Count = count() by InitiatingProcessFileName, FileName | where Count > 5; // At least 5 occurrences DeviceProcessEvents | where TimeGenerated > ago(1d) | join kind=leftanti ProcessRelationships on InitiatingProcessFileName, FileName | where InitiatingProcessFileName !in ("explorer.exe", "svchost.exe") | project TimeGenerated, DeviceName, Parent = InitiatingProcessFileName, Child = FileName, ProcessCommandLine | summarize Count = count(), Devices = make_set(DeviceName, 5) by Parent, Child | order by Count asc
KQL Hunt: First-time connections to external IPs
// Historical IPs seen in last 30 days let HistoricalIPs = DeviceNetworkEvents | where TimeGenerated between (ago(30d) .. ago(1d)) | where RemoteIPType == "Public" | distinct RemoteIP; // New IPs in last 24 hours DeviceNetworkEvents | where TimeGenerated > ago(1d) | where RemoteIPType == "Public" | join kind=leftanti HistoricalIPs on RemoteIP | summarize FirstSeen = min(TimeGenerated), ConnectionCount = count(), Devices = dcount(DeviceName), Processes = make_set(InitiatingProcessFileName, 5) by RemoteIP | where ConnectionCount > 10 // Focus on active connections | order by ConnectionCount desc
Key Takeaways
  • leftanti join is powerful for finding anomalies vs baseline
  • Time series analysis detects spikes and beaconing patterns
  • Always filter on time first for performance
  • Use has instead of contains when possible
  • Coefficient of variation < 0.2 indicates regular intervals (C2)
  • Build baselines from historical data, then compare recent activity
  • Save complex queries as functions for reuse
