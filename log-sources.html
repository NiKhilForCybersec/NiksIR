Log Priority Matrix

Not all logs are equally valuable. Focus collection efforts on high-value sources first.

Priority Log Sources Why Essential
Critical EDR telemetry, Authentication logs, Firewall deny logs Core visibility for threat detection
High DNS logs, Proxy/web logs, Email gateway, VPN logs Network visibility and lateral movement
Medium Windows Security events, Cloud audit logs, DHCP Context and attribution
Lower Application logs, Database logs, Printer logs Specific use cases, compliance
Coverage vs. Cost

SIEM licensing is often based on data volume. Prioritize high-value logs and consider filtering or sampling verbose logs. More data isn't always better if it's noise.

Endpoint Logs

EDR Telemetry â–¼

The richest source of endpoint visibility. Captures process, file, network, and registry activity.

Data TypeUse Case
Process creationExecution analysis, parent-child chains
Network connectionsC2 detection, lateral movement
File eventsMalware drops, data staging
Registry changesPersistence detection
DLL loadsInjection, hijacking

Examples: Microsoft Defender for Endpoint, CrowdStrike, SentinelOne, Carbon Black

Windows Security Events â–¼

Native Windows logging. Essential if no EDR or for additional context.

Key Event IDs:

  • 4624/4625: Logon success/failure
  • 4688: Process creation (with command line if enabled)
  • 4672: Special privileges assigned
  • 4720/4726: User account created/deleted
  • 7045: Service installed

See full Windows Event ID reference â†’

Sysmon â–¼

Free Microsoft tool that enhances Windows logging significantly.

Key Sysmon Events:

  • Event 1: Process creation (with hashes, parent info)
  • Event 3: Network connection
  • Event 8: CreateRemoteThread (injection)
  • Event 10: Process access (credential theft)
  • Event 11: File creation
  • Event 22: DNS query

Highly recommended for environments without EDR

PowerShell Logging â–¼

Critical for detecting PowerShell-based attacks (very common).

Enable These:

  • Module Logging (4103): Detailed command logging
  • Script Block Logging (4104): Full script content
  • Transcription: Full session recording to file

Logs obfuscated scripts in deobfuscated form!

Network Logs

Source What It Contains Key Use Cases
Firewall Allow/deny decisions, source/dest IP, ports Perimeter visibility, blocked attacks, C2 detection
DNS Query/response, domain names, client IPs C2 detection, DNS tunneling, malware domains
Web Proxy URLs, user agent, response codes, bytes Web-based attacks, C2, data exfiltration
VPN User authentication, source IPs, session duration Unauthorized access, impossible travel
NetFlow/IPFIX Traffic metadata (src/dst, bytes, packets) Baseline deviations, lateral movement, exfil
IDS/IPS Signature matches, anomalies Known attack patterns, exploitation attempts
DHCP IP-to-MAC mappings, lease times Asset identification, attribution
DNS is Gold

DNS logs are one of the highest-value log sources. Nearly all network activity involves DNS, malware must resolve C2 domains, and DNS tunneling is a common exfiltration method.

Identifying Log Gaps

Ask these questions to find visibility gaps:

  • Can I see who logged into any system? (Authentication)
  • Can I see what processes ran on endpoints? (Execution)
  • Can I see what network connections were made? (Network)
  • Can I see what files were accessed or transferred? (Data)
  • Can I correlate activity to a specific user? (Attribution)
  • Can I see activity on all critical assets? (Coverage)
  • Can I detect known attack patterns? (Detection)
  • Do I have logs going back far enough? (Retention)
Common Gaps
  • East-West traffic: Internal network flows often unmonitored
  • Linux systems: Often less logging than Windows
  • Cloud infrastructure: New cloud services may lack logging
  • OT/IoT devices: Limited or no logging capability
  • Encrypted traffic: TLS inspection gaps
Key Takeaways
  • Prioritize high-value logs: EDR, auth, firewall, DNS
  • EDR telemetry provides the richest endpoint visibility
  • DNS logs are underratedâ€”collect them
  • Cloud logs require explicit enablementâ€”verify they're on
  • Regularly assess log gaps against your threat model
  • Balance coverage with costâ€”filter verbose, low-value logs
