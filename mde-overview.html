What is Microsoft Defender for Endpoint?

Microsoft Defender for Endpoint (MDE) is Microsoft's enterprise endpoint security platform. It provides preventative protection, post-breach detection, automated investigation, and response capabilities for Windows, macOS, Linux, Android, and iOS devices.

Naming Note

MDE was previously known as "Microsoft Defender ATP" (MDATP), "Windows Defender ATP" (WDATP), and is part of "Microsoft 365 Defender" (now "Microsoft Defender XDR").

Key Capabilities

Endpoint Detection & Response (EDR) â–¼

Core detection and investigation capabilities:

  • Behavioral Sensors: Collect endpoint telemetry (process, file, network, registry)
  • Cloud Analytics: ML-based detection and threat intelligence
  • Alert Generation: Security alerts with severity and MITRE mapping
  • Incident Correlation: Group related alerts into incidents
  • Advanced Hunting: KQL-based threat hunting
  • Timeline: Chronological view of device activity
Next-Generation Antivirus â–¼

Built-in Microsoft Defender Antivirus with cloud-powered protection:

  • Real-time Protection: On-access scanning of files and processes
  • Cloud-delivered Protection: Rapid signature updates, ML analysis
  • Behavior Monitoring: Detect suspicious process behavior
  • Exploit Protection: Mitigate memory-based attacks
  • Attack Surface Reduction: Block risky behaviors (macros, scripts)
  • Network Protection: Block connections to malicious domains
Threat & Vulnerability Management (TVM) â–¼

Continuous vulnerability assessment and prioritization:

  • Software Inventory: Discover installed applications
  • Vulnerability Assessment: Identify known CVEs
  • Security Recommendations: Prioritized remediation guidance
  • Exposure Score: Organization-wide risk metric
  • Configuration Assessment: Security baseline compliance
Automated Investigation & Response (AIR) â–¼

Automated playbooks to investigate and remediate alerts:

  • Auto-investigation: Automatically analyze alerts and collect evidence
  • Remediation Actions: Quarantine files, stop processes, isolate devices
  • Approval Workflows: Require analyst approval for sensitive actions
  • Investigation Packages: Collect forensic data from devices
Response Actions â–¼

Manual response capabilities for analysts:

  • Device Isolation: Network isolate while maintaining MDE connection
  • Restrict App Execution: Only allow Microsoft-signed apps
  • Run Antivirus Scan: Trigger on-demand scan
  • Collect Investigation Package: Gather forensic artifacts
  • Initiate Live Response: Remote shell access to device
  • Stop and Quarantine File: Block and quarantine specific file

Key Data Tables

MDE data is available in Advanced Hunting using KQL. Key tables include:

Table Contains Common Use Cases
DeviceProcessEvents Process creation and termination Execution analysis, parent-child chains
DeviceNetworkEvents Network connections C2 detection, lateral movement
DeviceFileEvents File creation, modification, deletion Malware drops, data staging
DeviceRegistryEvents Registry changes Persistence detection
DeviceLogonEvents User logons to devices Authentication analysis
DeviceImageLoadEvents DLL loading DLL hijacking, injection
DeviceEvents Miscellaneous events (AMSI, etc.) Script content, security events
DeviceInfo Device inventory and details Asset information
AlertInfo Alert details Alert analysis
AlertEvidence Evidence associated with alerts Alert investigation
KQL Sample query: Process activity on device
DeviceProcessEvents | where TimeGenerated > ago(1h) | where DeviceName =~ "WORKSTATION01" | project TimeGenerated, FileName, ProcessCommandLine, InitiatingProcessFileName, AccountName | order by TimeGenerated desc | take 100

Alert Investigation Checklist

  • Read alert title and descriptionâ€”what is being detected?
  • Check alert severity and MITRE ATT&CK mapping
  • Review the process tree in alert details
  • Check the device timeline for context before/after alert
  • Identify affected device and user
  • Check if file hash is known malware (VT, MDE file page)
  • Look for related alerts on same device or user
  • Determine if this is True Positive, Benign TP, or False Positive

Identity threat detection for Active Directory

  • Lateral movement detection
  • Credential attacks
  • AD reconnaissance

Microsoft Defender for Office 365

Email and collaboration security

  • Phishing protection
  • Malicious attachment blocking
  • Safe Links

Microsoft Defender for Cloud Apps

Cloud application security (CASB)

  • Shadow IT discovery
  • App governance
  • Data protection

Microsoft Sentinel

Cloud-native SIEM

  • Central log aggregation
  • Cross-platform correlation
  • SOAR capabilities
