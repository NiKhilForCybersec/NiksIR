What is MITRE ATT&CK?

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) is a globally accessible knowledge base of adversary tactics and techniques based on real-world observations. It provides a common language for describing attacker behavior and helps defenders map detections to threat activities.

Tactics Overview (Kill Chain)

ATT&CK tactics represent the adversary's tactical goals during an attack. They answer "why" an attacker is performing an action.

ID Tactic Description Example Techniques
TA0043 Reconnaissance Gathering information to plan attack Active scanning, Search open websites
TA0042 Resource Development Establishing resources for operations Acquire infrastructure, Develop capabilities
TA0001 Initial Access Getting into the network Phishing, Exploit public-facing app, Valid accounts
TA0002 Execution Running malicious code PowerShell, Command line, Scripting
TA0003 Persistence Maintaining foothold Registry run keys, Scheduled tasks, Services
TA0004 Privilege Escalation Getting higher-level permissions Exploitation, Token manipulation, UAC bypass
TA0005 Defense Evasion Avoiding detection Disable security tools, Obfuscation, Log deletion
TA0006 Credential Access Stealing credentials Credential dumping, Keylogging, Brute force
TA0007 Discovery Learning about the environment Account discovery, Network scanning, File search
TA0008 Lateral Movement Moving through the network Remote services, Pass-the-hash, RDP
TA0009 Collection Gathering target data Archive data, Screen capture, Clipboard data
TA0011 Command & Control Communicating with compromised systems Web protocols, DNS, Encrypted channel
TA0010 Exfiltration Stealing data Exfil over C2, Exfil over web, Scheduled transfer
TA0040 Impact Manipulate, interrupt, or destroy systems Data encryption (ransomware), Defacement, Wiper

Using ATT&CK in SOC Operations

Alert Enrichment

Map alerts to ATT&CK techniques to understand:

  • What stage of attack is this?
  • What might come next?
  • What related activity should we look for?

Detection Gaps

Use ATT&CK to identify coverage gaps:

  • Map existing rules to techniques
  • Identify techniques without detection
  • Prioritize detection development

Threat Hunting

Structure hunts around ATT&CK:

  • Hunt for specific techniques
  • Validate detection coverage
  • Test against known threat groups

Incident Analysis

Map incidents to ATT&CK for:

  • Complete attack timeline
  • Identify scope and impact
  • Improve future detection
