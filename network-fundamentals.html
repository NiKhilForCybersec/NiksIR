OSI Model Quick Reference

Understanding the OSI model helps identify where attacks occur and what logs to examine.

SOC Focus

Most SOC work happens at Layers 3-7. Layer 3-4 (Network/Transport) for firewall logs, Layer 7 (Application) for proxy, email, and web application logs.

Zone Violation Alerts

Traffic that crosses zone boundaries unexpectedly is suspicious. Examples: DMZ server connecting to internal database directly, workstation accessing management network, guest network reaching internal servers.

Network Security Devices

Firewall ‚ñº

Controls traffic flow between networks based on rules.

Types:

  • Packet Filter: Basic L3/L4 filtering (src/dst IP, port)
  • Stateful: Tracks connection state
  • Next-Gen (NGFW): Application awareness, IPS, TLS inspection

SOC Use:

  • Review blocked/allowed connections
  • Identify suspicious outbound traffic
  • Request blocks for IOCs

Vendors: Palo Alto, Fortinet, Cisco, Check Point, Azure Firewall

IDS/IPS ‚ñº

Intrusion Detection/Prevention System - monitors traffic for threats.

Difference:

  • IDS: Detect and alert only (passive)
  • IPS: Detect and block (inline)

Detection Methods:

  • Signature-based: Known attack patterns
  • Anomaly-based: Deviation from baseline

Vendors: Snort, Suricata, Cisco, Palo Alto (built into NGFW)

Web Proxy ‚ñº

Intermediary for web traffic - filtering, logging, caching.

Functions:

  • URL filtering and categorization
  • Malware scanning
  • TLS/SSL inspection
  • Data loss prevention

SOC Use:

  • Investigate web activity for users
  • Identify C2 communication
  • Block malicious URLs

Vendors: Zscaler, Netskope, Broadcom (Symantec), McAfee

Email Gateway ‚ñº

Filters inbound/outbound email for threats and policy.

Functions:

  • Spam filtering
  • Malware/attachment scanning
  • Phishing detection
  • URL rewriting/sandboxing
  • DLP for outbound email

Vendors: Microsoft Defender for O365, Proofpoint, Mimecast, Cisco

NAC (Network Access Control) ‚ñº

Controls which devices can connect to the network.

Functions:

  • Device authentication (802.1X)
  • Posture assessment (is device compliant?)
  • Guest network management
  • VLAN assignment based on policy

Vendors: Cisco ISE, Aruba ClearPass, Forescout

Traffic Types & Direction

Ingress (Inbound)

Traffic entering your network from external sources.

  • Legitimate: Users accessing public services
  • Suspicious: Scanning, exploitation attempts

Egress (Outbound)

Traffic leaving your network to external destinations.

  • Legitimate: Web browsing, email
  • Suspicious: C2, data exfiltration

East-West (Lateral)

Traffic between internal systems.

  • Legitimate: Server-to-server, file shares
  • Suspicious: Lateral movement, scanning
SOC Priority

Egress traffic often reveals compromise. Attackers must communicate out for C2 or exfiltration. East-West traffic shows post-compromise activity and lateral movement.

What to Look For

Key indicators in network traffic that warrant investigation:

Indicator Why It Matters Example
Unusual ports May indicate tunneling or backdoors HTTP on port 8443, SSH on port 2222
High-volume transfers Potential data exfiltration 10GB upload from workstation
Beaconing patterns C2 communication Connection every 60 seconds to same IP
DNS anomalies DNS tunneling, DGA domains Long subdomain queries, high query volume
Rare destinations Potential C2 infrastructure Connections to IPs no one else visits
Protocol mismatches Tunneling, evasion Non-HTTP traffic on port 80/443
Internal scanning Reconnaissance, lateral movement Workstation hitting many internal IPs
After-hours activity Suspicious timing Large transfers at 3 AM
Key Takeaways
  • Understand the OSI model to identify attack layers
  • Know your network zones and expected traffic patterns
  • Firewall = traffic control, IDS/IPS = threat detection
  • Egress and east-west traffic often reveal compromise
  • Look for anomalies: unusual ports, volumes, timing, destinations
