P1 CRITICAL PB-02 Emergency Response

Ransomware Indicators

Signs that may indicate active ransomware:

Clear Indicators

  • Ransom note files (README.txt, DECRYPT.txt)
  • Changed desktop wallpaper with ransom message
  • Files with new extensions (.encrypted, .locked)
  • Users unable to open files
  • Mass file encryption alerts

Early Warning Signs

  • Unusual high CPU/disk activity
  • Shadow copy deletion (vssadmin)
  • Backup deletion attempts
  • Mass file modifications
  • Suspicious PowerShell/cmd activity
  • Lateral movement indicators
KQL Detect ransomware indicators
// Shadow copy deletion (common ransomware behavior) DeviceProcessEvents | where TimeGenerated > ago(1h) | where ProcessCommandLine has_any ( "vssadmin delete shadows", "wmic shadowcopy delete", "bcdedit /set", "wbadmin delete catalog" ) | project TimeGenerated, DeviceName, ProcessCommandLine, InitiatingProcessFileName, AccountName

Immediate Actions (First 15 Minutes)

ğŸš¨ Do These NOW

Execute in order. Speed is critical to prevent spread.

0-2 min
1. ESCALATE IMMEDIATELY
  • Call SOC Lead (don't just email/message)
  • Notify IR Team via emergency channel
  • Alert IT Operations
  • Trigger incident bridge/war room if available
2-5 min
2. ISOLATE AFFECTED SYSTEMS
  • Network isolate confirmed infected hosts via EDR
  • Do NOT power off (preserves memory evidence)
  • Disconnect from network but leave powered on
  • Block known malicious IPs/domains at firewall
5-10 min
3. STOP THE SPREAD
  • Identify and isolate any systems showing similar behavior
  • Consider isolating entire network segments if spreading fast
  • Disable compromised accounts
  • Block lateral movement ports if needed (SMB 445, RDP 3389)
10-15 min
4. PRESERVE EVIDENCE
  • Screenshot ransom notes
  • Collect investigation package from EDR
  • Note affected hostnames, IPs, users
  • Capture any IOCs (file hashes, C2 IPs, domains)

Containment Actions

Network Containment Options

Action When to Use Impact
Isolate Host (EDR) First choice for individual systems Low - affects only target host
VLAN Isolation Segment appears compromised Medium - affects network segment
Block SMB/RDP Lateral movement detected Medium - affects remote admin
Internet Disconnect C2 communication ongoing High - affects business operations
Full Network Shutdown Widespread, rapid encryption Critical - executive decision required
Containment Authority

Individual host isolation: L2 analyst can execute
Segment isolation: SOC Lead approval
Full network actions: IR Lead/CISO approval required

Immediate Blocks

  • Block known ransomware C2 IPs at perimeter firewall
  • Block malicious domains at DNS/proxy
  • Add file hashes to EDR block list
  • Disable compromised user accounts
  • Reset credentials for any accessed admin accounts

Scope Assessment

Rapidly determine the extent of the incident:

KQL Find all hosts with ransomware file hash
let ransomwareHash = "abc123..."; // SHA256 of ransomware DeviceFileEvents | where TimeGenerated > ago(24h) | where SHA256 == ransomwareHash | summarize FirstSeen = min(TimeGenerated), LastSeen = max(TimeGenerated), FileCount = count() by DeviceName | order by FirstSeen asc
KQL Find hosts with mass file modifications
DeviceFileEvents | where TimeGenerated > ago(2h) | where ActionType in ("FileModified", "FileRenamed") | summarize FileCount = count() by DeviceName, bin(TimeGenerated, 5m) | where FileCount > 500 // Unusual volume | order by FileCount desc

Scope Checklist

  • How many endpoints are affected?
  • Are servers affected? Which ones?
  • Are domain controllers affected? (Critical escalation)
  • Are backups affected or accessible?
  • What data/systems are encrypted?
  • Is encryption still active or stopped?
  • What is the ransomware family? (Check ransom note, extensions)

Investigation Focus Areas

Initial Access Vector â–¼

Determine how attackers got in:

  • Phishing: Check email logs for malicious attachments/links
  • Exploit: Check for vulnerable public-facing services (VPN, RDP)
  • Credential Theft: Check for brute force or credential stuffing
  • Supply Chain: Check for compromised software updates
Attacker Dwell Time â–¼

Most ransomware actors are in the network days/weeks before encrypting:

  • When did initial access occur?
  • What reconnaissance was performed?
  • What data was accessed before encryption?
  • Was data exfiltrated? (Double extortion)
Lateral Movement Path â–¼

Trace how ransomware spread:

  • What accounts were used for lateral movement?
  • What tools were used (PsExec, WMI, RDP)?
  • Which systems were accessed?
  • How did they get domain admin?
Ransomware Identification â–¼

Identify the ransomware family:

  • Check ransom note text and file names
  • Check encrypted file extensions
  • Upload sample to ID Ransomware
  • Check for known decryptors at No More Ransom

Recovery Planning

Recovery Decisions Require Executive Input

Major recovery decisions (restore from backup, rebuild, negotiate) require CISO/executive approval. SOC's role is to provide information and options.

Recovery Options

Option Prerequisites Timeframe
Restore from Backup Clean, recent backups available; backups not encrypted Hours to days
Use Decryptor Known ransomware with available decryptor Hours
Rebuild Systems No backups; clean images available Days to weeks
Pay Ransom Last resort; executive/legal decision; no guarantee Days (negotiation)

Before Recovery

  • Ensure attacker access is fully removed
  • Patch exploited vulnerabilities
  • Reset all potentially compromised credentials
  • Verify backup integrity before restore
  • Plan for enhanced monitoring post-recovery

Critical Don'ts

âŒ Do NOT:

  • Do NOT power off systems - Preserves memory evidence, encryption keys may be in RAM
  • Do NOT pay ransom immediately - Executive/legal decision, may be illegal, no guarantee of decryption
  • Do NOT restore backups to infected network - They'll get encrypted too
  • Do NOT delete ransomware files - Needed for identification and potential decryption
  • Do NOT communicate ransom demands via work email - Attackers may be monitoring
  • Do NOT assume it's contained - Attackers often have multiple access points
  • Do NOT rush recovery - Incomplete eradication leads to re-infection
Playbook Summary
  1. ESCALATE - Immediately notify leadership
  2. ISOLATE - Contain affected systems (don't power off)
  3. STOP SPREAD - Block lateral movement and C2
  4. PRESERVE - Collect evidence and IOCs
  5. ASSESS - Determine scope of encryption
  6. INVESTIGATE - Find initial access and dwell time
  7. RECOVER - Plan recovery with leadership approval
