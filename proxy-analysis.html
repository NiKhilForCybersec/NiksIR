What is a Web Proxy?

A web proxy acts as an intermediary between users and the internet. All web traffic flows through the proxy, enabling inspection, filtering, logging, and policy enforcement.

Why Use a Proxy?

Security

  • Block malicious websites
  • Prevent malware downloads
  • Inspect encrypted traffic (SSL)
  • Detect data exfiltration

Policy Enforcement

  • Block inappropriate content
  • Enforce acceptable use
  • Limit bandwidth usage
  • Restrict cloud apps

Visibility

  • Log all web activity
  • User attribution
  • Application identification
  • Forensic evidence

Proxy Types

Type Description Examples
Forward Proxy Sits in front of clients, handles outbound requests Squid, Bluecoat
Reverse Proxy Sits in front of servers, handles inbound requests Nginx, HAProxy, CloudFlare
Transparent Proxy Intercepts traffic without client configuration Network-level interception
Cloud Proxy (SWG) Cloud-based secure web gateway Zscaler, Netskope, Umbrella
Modern Trend: Cloud Proxies

With remote work, many organizations use cloud-based Secure Web Gateways (SWG) like Zscaler or Netskope. These provide proxy functionality for users anywhere, not just on-premises.

Threat Detection Use Cases

Malware Download Attempts ‚ñº

Indicators:

  • Category: Malware, Phishing, Uncategorized
  • File types: .exe, .dll, .scr, .ps1, .bat
  • Newly registered domains
  • Low reputation scores

Action:

Check if blocked. If allowed, investigate endpoint for compromise.

Command & Control (C2) Communication ‚ñº

Indicators:

  • Repeated connections to same domain
  • Regular timing intervals (beaconing)
  • POST requests with encoded data
  • Unusual user-agents
  • Category: Uncategorized, newly seen
Data Exfiltration ‚ñº

Indicators:

  • Large POST requests (unusual byte counts)
  • Uploads to cloud storage (Dropbox, Google Drive)
  • Traffic to paste sites (Pastebin)
  • Unusual protocols over port 443
Crypto Mining ‚ñº

Indicators:

  • Connections to mining pools
  • High-volume WebSocket connections
  • Category: Crypto Mining
  • Known mining domains
Proxy Bypass Attempts ‚ñº

Indicators:

  • Category: Anonymizer, VPN, Proxy Avoidance
  • Connections to known VPN/Tor services
  • Non-standard ports for HTTP traffic
  • Attempts to reach IP addresses directly

Why It Matters:

Users bypassing proxy may be hiding malicious activity or violating policy.

Investigation Queries

KQL Blocked malware downloads
CommonSecurityLog | where TimeGenerated > ago(24h) | where DeviceVendor == "Zscaler" // or your proxy vendor | where Activity == "Blocked" | where DeviceCustomString1 has_any ("Malware", "Phishing", "Botnet") | project TimeGenerated, SourceUserName, SourceIP, RequestURL, DeviceCustomString1, DeviceAction | order by TimeGenerated desc
KQL Large uploads (potential exfiltration)
CommonSecurityLog | where TimeGenerated > ago(24h) | where RequestMethod == "POST" | where SentBytes > 10000000 // 10 MB | project TimeGenerated, SourceUserName, SourceIP, RequestURL, SentBytes, DeviceCustomString1 | order by SentBytes desc
KQL User activity to specific domain
let suspiciousDomain = "suspicious.com"; CommonSecurityLog | where TimeGenerated > ago(7d) | where RequestURL contains suspiciousDomain | summarize RequestCount = count(), FirstSeen = min(TimeGenerated), LastSeen = max(TimeGenerated) by SourceUserName, SourceIP | order by RequestCount desc

Key Takeaways
  • Proxies provide visibility into all web traffic with user attribution
  • Check if blocked downloads reached the endpoint
  • Look for beaconing patterns (regular intervals to same destination)
  • Large uploads, especially to cloud storage, may indicate exfiltration
  • Proxy bypass attempts are often suspicious
  • Correlate proxy logs with endpoint data for full picture
