What is Microsoft Sentinel?

Microsoft Sentinel is a cloud-native Security Information and Event Management (SIEM) and Security Orchestration, Automation, and Response (SOAR) solution. It provides intelligent security analytics across your enterprise.

SIEM
Security Information & Event Management
SOAR
Security Orchestration & Automation
UEBA
User & Entity Behavior Analytics
TI
Threat Intelligence Integration

Key Capabilities

  • Collect - Ingest data from any source at cloud scale
  • Detect - Analytics rules, ML, and threat intelligence for detection
  • Investigate - KQL hunting, investigation graphs, notebooks
  • Respond - Automation playbooks with Logic Apps

Log Analytics Workspace

Sentinel is built on top of Azure Log Analytics. The workspace is where all your security data is stored and queried using Kusto Query Language (KQL).

Workspace Basics
  • Data stored in tables (e.g., SecurityEvent, SigninLogs)
  • Retention configurable (default 90 days, up to 2 years)
  • Query using KQL in Logs blade
  • Cost based on data ingestion volume (GB/day)

Incident Investigation Checklist

  • Read incident title, description, and severity
  • Review all related alerts and their details
  • Check entities involved (users, hosts, IPs)
  • Use investigation graph for visual analysis
  • Query relevant tables for additional context
  • Check entity pages for historical behavior
  • Determine if True Positive or False Positive
  • Document findings and take appropriate action
Key Takeaways
  • Sentinel = SIEM + SOAR built on Azure Log Analytics
  • Data flows: Sources ‚Üí Connectors ‚Üí Workspace ‚Üí Analytics ‚Üí Incidents
  • Master the Incidents and Logs blades for daily work
  • KQL is your primary investigation language
  • Use Content Hub for pre-built solutions and rules
  • Integrates natively with M365 Defender for XDR capabilities
