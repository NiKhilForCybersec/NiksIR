Ticket Lifecycle

Every security alert or incident follows a lifecycle from creation to closure. Understanding this flow helps manage workload and ensures nothing falls through the cracks.

Ticket vs. Incident

Alert/Ticket: Individual detection requiring investigation
Incident: Confirmed security event (may contain multiple related tickets)

Status Management

Status Meaning When to Use
New/Open Awaiting analyst assignment Initial state, no one working on it yet
Assigned Analyst has taken ownership You've claimed the ticket
In Progress Active investigation underway You're actively working on it
Pending/On Hold Waiting for external input Waiting for user response, IT action, etc.
Escalated Transferred to higher tier L1 â†’ L2, L2 â†’ L3, or IR team
Resolved Investigation complete, actions taken Work done, documented, verified
Closed No further action needed Final state after resolution verified
Keep Status Updated

An accurate status helps the team understand workload. If you step away from a ticket, update the status. Nothing is worse than a ticket showing "In Progress" for hours with no updates.

Queue Prioritization

Priority Order

  1. P1 Critical - Always first, no exceptions
  2. P2 High - After P1s are handled
  3. SLA at risk - Any ticket approaching SLA breach
  4. P3 Medium - Standard work
  5. P4 Low - When queue is clear

Queue Management Tips

Start of Shift

  • Review handoff notes
  • Check for P1/P2 alerts
  • Identify SLA-at-risk tickets
  • Scan for patterns in queue

During Shift

  • Work highest priority first
  • Don't cherry-pick easy tickets
  • Group similar alerts for efficiency
  • Update status regularly

End of Shift

  • Document in-progress work
  • Update ticket statuses
  • Prepare handoff notes
  • Flag anything time-sensitive

Ticket Documentation

Essential Ticket Fields

Field Content
Summary/Title Clear, specific description (not just alert name)
Priority P1/P2/P3/P4 based on severity and impact
Category Malware, phishing, account compromise, etc.
Affected Assets Hostnames, IPs, user accounts
Investigation Notes What you found, what you checked
Actions Taken Containment, remediation steps
Verdict TP, BTP, FP, Inconclusive
IOCs IPs, domains, hashes, file paths

Common Ticket Issues

Issue Problem Solution
Stale tickets Tickets sitting with no updates Set calendar reminders, use "Pending" status with reason
Ticket hoarding Analyst assigns many tickets but doesn't work them Only assign what you'll work now
Insufficient documentation Next analyst can't understand what was done Document as you go, use templates
Wrong priority P3 treated as P1 or vice versa Follow classification guidelines
Duplicate tickets Same issue has multiple tickets Search before creating, link related tickets
Premature closure Closed before verified complete Verify actions worked before closing
