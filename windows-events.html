Event ID Ranges

Authentication Events

Critical events for tracking user logon activity, authentication attempts, and session management.

Event ID Event Name Description Security Value
4624 Successful Logon Account successfully logged on High - Track successful access, correlate with failures
4625 Failed Logon Account failed to log on Critical - Brute force, password spray detection
4634 Logoff Account logged off Medium - Session duration analysis
4647 User Initiated Logoff User initiated logoff Low - Normal user behavior
4648 Explicit Credentials Logon attempted using explicit credentials High - Lateral movement indicator
4768 Kerberos TGT Request Kerberos authentication ticket (TGT) was requested High - AS-REP roasting detection
4769 Kerberos Service Ticket Kerberos service ticket (TGS) was requested Critical - Kerberoasting detection
4771 Kerberos Pre-Auth Failed Kerberos pre-authentication failed High - Password attacks
4776 NTLM Authentication DC attempted to validate credentials (NTLM) High - Pass-the-hash detection

Logon Type Name Description

3NetworkNetwork logon (e.g., accessing share)
4BatchScheduled task execution
5ServiceService startup
7UnlockWorkstation unlock
8NetworkCleartextNetwork logon with cleartext creds
9NewCredentialsRunAs with /netonly
10RemoteInteractiveRDP logon
11CachedInteractiveLogon with cached creds (offline)

Account Management Events

Track user and group changes - critical for detecting privilege escalation and persistence.

Event ID Event Name Description Security Value
4720 User Account Created A user account was created High - Persistence, unauthorized accounts
4722 User Account Enabled A user account was enabled Medium - Re-enabled dormant accounts
4723 Password Change Attempt User attempted to change their password Low - Normal activity
4724 Password Reset Attempt Attempt to reset password by admin Medium - Admin activity tracking
4725 User Account Disabled A user account was disabled Low - Track offboarding
4726 User Account Deleted A user account was deleted Medium - Evidence destruction?
4728 Member Added to Global Group Member added to security-enabled global group High - Privilege escalation
4732 Member Added to Local Group Member added to security-enabled local group High - Local admin addition
4735 Local Group Changed Security-enabled local group was changed Medium - Group modifications
4738 User Account Changed A user account was changed Medium - Account manipulation
4740 Account Locked Out A user account was locked out High - Brute force indicator
4756 Member Added to Universal Group Member added to security-enabled universal group Critical - Enterprise admin groups
Critical Groups to Monitor
  • Domain Admins - Full domain control
  • Enterprise Admins - Forest-wide control
  • Schema Admins - AD schema modifications
  • Administrators - Local admin access
  • Backup Operators - Can backup/restore files
  • Account Operators - Can manage most accounts

Process Events

Process creation and termination events are essential for detecting malicious execution.

Event ID Event Name Description Security Value
4688 Process Created A new process has been created Critical - Primary execution detection
4689 Process Exited A process has exited Low - Duration analysis
Enable Command Line Logging

Event 4688 requires "Include command line in process creation events" policy to capture full command lines. Without this, you only see the executable path, not the arguments.

Key Fields in Event 4688

  • NewProcessName: Full path of the created process
  • CommandLine: Command line arguments (if enabled)
  • ParentProcessName: Process that created this one
  • SubjectUserName: User context of execution
  • TokenElevationType: Elevation status (1=default, 2=elevated, 3=limited)

Privilege Use Events

Track when sensitive privileges are assigned or used - key for detecting privilege escalation.

Event ID Event Name Description Security Value
4672 Special Privileges Assigned Special privileges assigned to new logon High - Admin/sensitive logon detection
4673 Privileged Service Called A privileged service was called Medium - Sensitive operation tracking
4674 Privileged Operation Operation attempted on privileged object Medium - Object access patterns

Sensitive Privileges (Event 4672)

High-Risk Privileges

  • SeDebugPrivilege - Debug processes (credential theft)
  • SeTcbPrivilege - Act as part of OS
  • SeBackupPrivilege - Backup files (bypass access)
  • SeRestorePrivilege - Restore files
  • SeTakeOwnershipPrivilege - Take ownership

Medium-Risk Privileges

  • SeRemoteShutdownPrivilege - Remote shutdown
  • SeLoadDriverPrivilege - Load drivers
  • SeCreateTokenPrivilege - Create tokens
  • SeImpersonatePrivilege - Impersonate clients
  • SeEnableDelegationPrivilege - Delegation

4658 Handle Closed Handle to an object was closed Low - Duration tracking
4660 Object Deleted An object was deleted High - Evidence destruction
4663 Object Access Attempt Attempt to access an object High - File/registry access
4670 Permissions Changed Permissions on object were changed High - ACL manipulation

4699 Scheduled Task Deleted A scheduled task was deleted Medium - Cleanup activity

PowerShell Events

PowerShell logging is critical for detecting script-based attacks.

Event ID Log Description Security Value
4103 PowerShell/Operational Module Logging - Command execution High - Full command capture
4104 PowerShell/Operational Script Block Logging - Script content Critical - Full script content
4105 PowerShell/Operational Script Block Logging - Start Low - Script execution start
4106 PowerShell/Operational Script Block Logging - Stop Low - Script execution end
400 Windows PowerShell Engine state changed to Available Low - PowerShell started
800 Windows PowerShell Pipeline execution details Medium - Execution tracking
Enable PowerShell Logging

Script Block Logging (4104) captures the full content of PowerShell scripts, including decoded/decrypted content. This is essential for detecting encoded commands and obfuscated malware.

Sysmon Events

Sysmon provides enhanced visibility beyond native Windows logging. Install and configure Sysmon for comprehensive endpoint telemetry.

Event ID Event Name Description Security Value
1 Process Create Process creation with full command line and hashes Critical - Primary execution telemetry
2 File Creation Time Changed File creation time was modified (timestomping) High - Anti-forensics detection
3 Network Connection Network connection initiated by process High - C2, data exfil detection
5 Process Terminated Process terminated Low - Duration analysis
6 Driver Loaded Driver loaded into kernel High - Rootkit detection
7 Image Loaded DLL loaded by process Medium - DLL hijacking/injection
8 CreateRemoteThread Remote thread created in another process Critical - Process injection
10 Process Access Process opened another process Critical - LSASS access, credential theft
11 File Create File created or overwritten Medium - File drops, staging
12 Registry Event - Create/Delete Registry key/value created or deleted High - Persistence detection
13 Registry Event - Set Value Registry value modified High - Configuration changes
15 FileCreateStreamHash Alternate data stream created High - Hidden data storage
17 Pipe Created Named pipe created Medium - C2 communication
18 Pipe Connected Named pipe connection Medium - Lateral movement
22 DNS Query DNS query made by process High - C2, DNS tunneling
23 File Delete File deleted (archived) Medium - Evidence destruction
25 Process Tampering Process image changed (hollowing) Critical - Process hollowing
Sysmon Configuration

Use a curated Sysmon configuration like SwiftOnSecurity/sysmon-config or olafhartong/sysmon-modular for optimal coverage with reduced noise.
